# Keenon Cloud API Development Document V2.2.0

> **Version:** V2.2.0 (2025-02-20)  
> **Auth:** OAuth2 client credentials — all endpoints (except token) require `Authorization: bearer <access_token>`

---

## Domain Names

| Region | Domain |
|--------|--------|
| China (domestic) | `https://console.peanut.keenonrobot.com` |
| EU | `https://es.robotkeenon.com` |
| Japan | `https://cloud.robotkeenon.com` |

All paths below are relative to your region's domain (referred to as `api_domain`).

---

## Key Concepts

| Term | Description |
|------|-------------|
| client_id | Unique client identifier on the open platform, generated by Keenon |
| client_secret | Secret key paired with client_id. Cannot be recovered — contact Keenon to reset |
| access_token | Bearer token exchanged from client_id/client_secret. Validity extends with each successful request |
| storeId | Unique ID of a store (physical location) on the cloud |
| robotId / robotSn | Unique identifier for a robot (MAC address format, e.g. `AA:BB:CC:DD:EE:FF`) |
| sceneCode | Unique ID of a scene — used for point configuration within a store |

---

## Common Response Format

All endpoints (except token) return:

```json
{
  "code": 610000,
  "msg": "Request successful",
  "data": { }
}
```

### Return Codes

| Code | Meaning |
|------|---------|
| 610000 | Success |
| 610001 | Wrong username or password (client_id / client_secret) |
| 610401 | Token verification failed / expired |
| 610403 | Insufficient operation permission |
| 610500 | Server exception |
| 610601 | Request parameter exception |
| 617000 | Request IP restricted (risk control triggered) |

## 3.2 Common Interfaces

### Get access_token

```
POST /api/open/oauth/token
Content-Type: application/x-www-form-urlencoded
```

**Body Parameters:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| client_id | String | yes | Client ID |
| client_secret | String | yes | Client secret |
| grant_type | String | yes | Fixed value: `client_credentials` |

**Response:**

```json
{
  "access_token": "009a8332-634e-4886-9c99-324caac55166",
  "token_type": "bearer",
  "expires_in": 6616,
  "scope": "all"
}
```

| Field | Type | Description |
|-------|------|-------------|
| access_token | String | Authorization token |
| token_type | String | Fixed value `bearer` |
| expires_in | Integer | Remaining validity in seconds (default 7200s on first request) |
| scope | String | Valid domain (currently unused, value `all`) |

> **Notes:** Each valid request extends token validity. Repeated token requests do not reset expiration. Do not hardcode `expires_in` — parse it dynamically.

## Task Error Codes

| Code | Description |
|------|-------------|
| 300 | Remote call function not enabled on robot |
| 301 | Robot is being set up |
| 302 | Robot is executing a task and cannot be called |
| 303 | Robot refuses to execute the task |
| 304 | No corresponding task on robot |
| 305 | Robot cannot recognize the target point |
| 400 | Robot manually canceled the task |
| 401 | Corresponding task record not found on robot |
| 402 | Unknown exception on robot, task ended |
| 403 | Robot judged the task timed out |
| 500 | Server did not find an available robot |
| 501 | Task not updated for a long time, ended by background |
| 502 | Robot offline, task not updated |
| 503 | Robot has not registered for call service |

## Webhooks (Callbacks)

The user provides a webhook URL. Keenon pushes status changes to this URL as JSON POST requests. Each callback has a `bizType` field identifying the event type.

### Callback Event Types

| bizType | Description |
|---------|-------------|
| `CreateTask` | Remote call task status change |
| `RobotOnlineStatus` | Robot online/offline status change |
| `RobotOnlineType` | Robot network type change (Wi-Fi, 4G, etc.) |
| `RobotPowerInfo` | Battery/charge status update |
| `HotelOrderStatus` | Hotel dispatch order status change |
| `HotelRobotBizTask` | Hotel robot delivery task update |
| `RobotWorkState` | Working status / callable status change |
| `RobotPositionType` | Robot position and nearby point update |
| `RobotTaskState` | Remote task real-time status |
| `CleanRobotStatus` | Cleaning robot real-time status |
| `CleanRobotRechargeTask` | Cleaning robot recharge task receipt |
| `CleanRobotFinishTask` | Cleaning robot finish task receipt |
| `CleanRobotPauseTask` | Cleaning robot pause task receipt |
| `CleanStrategyTemporary` | Cleaning robot temporary task receipt |
| `AddCleanStrategy` | Cleaning robot create scheduling task receipt |
| `UpdateCleanStrategy` | Cleaning robot modify scheduling task receipt |
| `DeleteCleanStrategy` | Cleaning robot delete/enable/disable scheduling task receipt |
